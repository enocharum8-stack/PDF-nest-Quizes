Week 1 — Foundations

A library is an organized collection of resources (physical/digital) for access, education, and community support. Three types: Traditional (physical, in-person), Digital Library (organized digital collection, access via computer terminals/local network), and Virtual Library (a digital library linked to the Internet for remote access from anywhere).

Key idea: a digital library becomes virtual once linked to the Internet. Per Usman (2019), a virtual library provides a single interface/entry point with remote access. Gapen (1993) emphasizes "remote access to contents and services," combining onsite electronic collections with external networked sources.

Evolution timeline: 1965 Licklider envisioned digital libraries → 1968 MARC (Library of Congress) → 1971 Project Gutenberg (Michael Hart, first digital book) → 1991 arXiv.org → 1994 DLI (NSF/DARPA/NASA) → 1996 Google founded → 2000 DOI system → 2004 Google Books → 2008 DOAJ → 2011 HathiTrust.

Week 2 — Building & Architecture

Building a digital library: (1) acquisition of original digital content with author/copyright permission, (2) conversion of non-digital documents via OCR into an Institutional Digital Repository (IDR), (3) subscribing/linking to open-access or licensed content, (4) rights management and user authentication.

Services: online full-text databases, offline CD-ROM databases (useful where Internet access is limited), OPAC (Online Public Access Catalogue — search by author/title/keyword/call number), online reference services, and Internet access provision.

Alexandria Digital Library (ADL) — three-tier architecture: Client → Middleware → Server, hosted at UC Santa Barbara, focused on geographically referenced materials. Greenstone (GSDL) — built by University of Waikato with UNESCO, generates HTML pages from retrieved info, aimed at the developing world.

Week 3 — Technologies & Platforms

Library management software: Koha (open-source ILS), NewGenLib, CodeAchi, BiblioteQ, Handy Library Manager. Digital library platforms: DSpace, Greenstone, Eprints, Omeka.

Cloud computing benefits for libraries: cost reduction, enhanced accessibility, scalability/flexibility, collaboration/sharing, but raises data security/privacy concerns. AI assists with information retrieval, library operations management, and accessibility (chatbots, personalized recommendations).

Weeks 4–5 — Metadata

Metadata = descriptive data about content, not the content itself (author, date, file size, keywords). Standards: Dublin Core (15 core elements: creator, title, date, subject, etc. — broad resource description), MARC (library bibliographic cataloging/exchange standard), MODS (metadata syndication/harvesting for digital info packages).

Indexing = assigning keywords/subject terms for retrieval. Cataloging = creating metadata that describes/organizes resources for accessibility.

Weeks 6–7 — Preservation & Access

Digital Right Management (DRM) uses access control to prevent unauthorized use/protect IP. Open access = free, unrestricted research access vs. subscription-based = requires paid access. Cybersecurity protects systems/data from cyberattacks; ethics covers privacy and responsible data handling.

Weeks 8–9 — Virtual Libraries & Trends

Digital library = collection of digital objects in machine-readable formats; virtual library = emphasizes access/organization via internet links rather than the collection itself. Applications: academic (Google Scholar, IEEE Xplore), e-books (Kindle, Google Books), public apps (Libby, OverDrive), specialized (Project Gutenberg, Sci-Hub). Blockchain enhances security, transparency, and IP management via a distributed immutable ledger.

Cloud Computing — Core Concepts (NIST)

Five essential characteristics: On-Demand Self-Service, Broad Network Access, Resource Pooling (multi-tenant), Rapid Elasticity, Measured Service (utility billing). Originated conceptually in 1961 (John McCarthy); SaaS pioneered by Salesforce (1999); modern era began with AWS EC2 (2006), followed by Azure (2010) and GCP.

Service models (SPI): IaaS (provider manages hardware; e.g. AWS EC2), PaaS (provider manages infra+platform; e.g. Google App Engine), SaaS (fully managed apps; e.g. Microsoft 365). Emerging: FaaS/Serverless (AWS Lambda), DBaaS (Amazon RDS), AIaaS (SageMaker/Vertex AI).

Deployment models: Public (shared, low cost), Private (dedicated, high control), Hybrid (mix for sensitive + bursting workloads), Multi-Cloud (multiple public providers, ~90% of enterprises, avoids lock-in).

Technologies: Virtualization (hypervisors raise utilization from 10-20% to 60-80%), Containerization (Docker) + Kubernetes orchestration, Microservices, IaC (Terraform). Market leaders: AWS (~30-32%), Azure (~20-25%), GCP (~10-12%) — together ~65-70% share.

Security: Shared Responsibility Model — provider secures "of the cloud," customer secures "in the cloud" (responsibility decreases IaaS→PaaS→SaaS). Key tech: zero-trust, IAM, encryption (AES-256, TLS 1.3), confidential computing. Compliance: GDPR, CCPA, HIPAA, PCI DSS.
