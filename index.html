Week 1 — Overview of AI

  • AI = field of CS creating systems that perform tasks needing human intelligence
  • Core capabilities: learning, reasoning, problem-solving, perception, language understanding
  • Narrow AI excels at specific tasks; General AI not yet achieved
  • AI impacts: economy, healthcare, education, transportation, security, R&D

Week 2 — Generative AI

  • Generative AI: models that create new content (text, images, audio, video, 3D)
  • GAN: Generator + Discriminator compete; VAE: Encoder → latent space → Decoder
  • Transformer Networks: attention mechanism; powers LLMs
  • Diffusion Models: start with noise, gradually denoise to generate realistic data
  • Risks: deepfakes, misinformation, copyright issues

Week 3 — AI Applications

  • NLP: chatbots, translation, sentiment analysis, text summarization
  • Computer Vision: facial recognition, object detection, medical imaging (CNNs)
  • Robotics: industrial, logistics, surgical, AMRs
  • Sectors: healthcare X-ray analysis, finance fraud detection, agriculture drones

Week 4 — ML Foundations

  • Arthur Samuel (1959): coined ML; Tom Mitchell (1997): T, E, P definition
  • Traditional programming: Data + Rules → Answers; ML: Data + Answers → Rules (model)
  • Key terms: feature, label, dataset, training set, validation set, test set
  • Overfitting: memorises training data; Underfitting: too simple, poor on all data

Week 5 — Types of ML

  • Supervised: labelled data; Classification (discrete) + Regression (continuous)
  • Unsupervised: unlabelled data; Clustering (K-Means, DBSCAN), Dimensionality Reduction (PCA)
  • Reinforcement Learning: agent, environment, reward/penalty, policy; AlphaGo example
  • Binary classification = 2 classes; Multi-class = 3+ classes

Week 6 — Data in AI

  • "Garbage in, garbage out" — data quality = model quality
  • Data prep = 70–80% of ML project time
  • Issues: missing values, duplicates, outliers, inconsistent formatting, imbalanced data
  • Sources: databases, APIs, web scraping, sensors, surveys, public datasets

Week 7 — Intelligent Agents

  • Agent: perceives via sensors, acts via actuators
  • Types: Simple Reflex → Model-Based → Goal-Based → Learning Agent
  • Fully vs. partially observable environments; critic provides feedback in learning agents

Week 8 — ML Process

  • 5 steps: Problem Definition → Data Prep → Model Training → Evaluation → Deployment
  • Loss function quantifies error; Gradient Descent minimises it
  • Learning rate: step size; Hyperparameter tuning on validation set only
  • Classification metrics: Accuracy, Precision, Recall, F1; Regression: RMSE, MAE, R²
  • K-Fold Cross-Validation: more robust estimate; Data Leakage: serious pitfall

Week 9 — AI Algorithms

  • Rule-based systems: expert systems with if-then rules
  • Neural network layers: Input → Hidden → Output; activation functions add non-linearity
  • Naive Bayes: probabilistic, fast, great for text; KNN: classify by K nearest neighbours
  • Deep learning: many hidden layers, hierarchical feature learning

Week 10 — Ethics & Societal Impact

  • AI Ethics: moral compass; AI Governance: rules of the road
  • Concerns: bias, black box problem, privacy, job displacement, deepfakes, autonomous weapons
  • Principles: Beneficence, Fairness, Transparency, Accountability, Privacy, Human Control
  • XAI: Explainable AI; EU AI Act: risk-based regulatory framework

Week 11 — Careers & Trends

  • Roles: AI/ML Engineer, Data Scientist, AI Researcher, AI Ethicist, Robotics Engineer, NLP Specialist, Computer Vision Engineer
  • IBM AI Ladder: Collect → Organize → Analyze → Infuse
  • AI adoption considerations: business goals, data readiness, ethics, talent, ROI
